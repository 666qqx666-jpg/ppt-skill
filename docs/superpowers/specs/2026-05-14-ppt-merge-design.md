# PPT 合并 Skill 设计文档

## 概述

一个 Claude Code skill，接收用户提供的多个使用相同模板的 PPT 文件，按页面标题分组归类，将相同标题的页面排列在一起，生成一个合并后的 PPT 文件。

**技术栈：** python-pptx
**实现方案：** 标题索引分组 + 幻灯片原样复制

## 核心逻辑

### 输入/输出

- **输入：** 2 个或以上 `.pptx` 文件路径（用户以列表形式提供）
- **输出：** 保存至 `/Users/qqx/my_code_cursor/lq/output/merged_<YYYYMMDD_HHMMSS>.pptx`
- 成功时返回输出文件路径，失败时返回具体错误信息

### 处理流程

1. **验证输入** — 检查所有文件是否存在、格式是否为有效 .pptx、文件数量 ≥ 2
2. **打开文件** — 用 python-pptx 加载所有 PPT
3. **提取标题** — 从每个 PPT 的每张幻灯片中提取并归一化标题
4. **建立标题索引** — 以第一个 PPT 的页面顺序为骨架，构建全局标题排序
5. **处理新标题** — 其他 PPT 中出现的新标题按其在源文件中的相对位置插入索引
6. **分组复制** — 按标题索引顺序输出：封面 + 各标题组（组内按 PPT 输入顺序）
7. **保存** 合并后的演示文稿到输出目录

### 标题提取

优先级顺序：
1. 使用占位符类型 `TITLE` 或 `CENTER_TITLE` 的文本
2. 回退到幻灯片中最上方的文本框
3. 再回退到最大面积的文本框

### 标题归一化（模糊匹配）

对提取的标题文本执行以下归一化操作，用于分组比较：

1. 去除首尾空白
2. 全角字符转半角
3. 去除前导序号（如 `1.`、`一、`、`(1)`、`①` 等）
4. 统一转小写（处理中英文混用场景）
5. 压缩连续空格为单个空格

归一化仅用于匹配比较，不修改幻灯片中的实际标题文本。

### 分组排序算法

```
输入: ppts = [PPT_1, PPT_2, ..., PPT_N]
输出: 按标题分组排序的幻灯片列表

1. title_order = []  # 有序标题列表
2. groups = {}       # {归一化标题: [幻灯片列表]}

3. 遍历 PPT_1 的第 2 页至第 N 页:
     提取标题 → 归一化 → 追加到 title_order
     将该幻灯片加入 groups[归一化标题]

4. 对 PPT_2 到 PPT_N 中的每个 PPT:
     遍历第 2 页至第 N 页:
       提取标题 → 归一化
       if 标题 in title_order:
         将幻灯片追加到 groups[该标题]
       else:
         根据该页在源 PPT 中前后已知标题的位置，
         将新标题插入 title_order 的对应位置
         创建 groups[新标题] = [该幻灯片]

5. 输出:
     封面 = PPT_1 的第 1 页
     内容 = 按 title_order 顺序，依次输出 groups[标题] 中的所有幻灯片
```

### 新标题插入位置算法

当 PPT-B 中出现一个标题 X 不在 `title_order` 中时：

1. 找到 X 在 PPT-B 中的前一个标题 `prev`（已存在于 `title_order` 中的最近标题）
2. 找到 X 在 PPT-B 中的后一个标题 `next`（已存在于 `title_order` 中的最近标题）
3. 将 X 插入到 `prev` 之后（如果 `prev` 存在），否则插入到 `next` 之前
4. 如果前后都没有已知标题，追加到 `title_order` 末尾

### 幻灯片复制策略

因为所有 PPT 使用同一模板，合并操作的核心是原样复制幻灯片，不修改任何内容或样式：

- 深拷贝幻灯片 XML（`sld.xml`）
- 在目标文件中重新注册所有关联资源（图片、图表、嵌入对象等 rels）
- 保留幻灯片的版式引用（slide layout）——模板相同，版式定义一致
- 复制幻灯片备注页（如果存在）

### 封面处理

- 直接复制第一个 PPT 的第 1 页作为合并结果的封面
- 其他 PPT 的封面（第 1 页）丢弃，不参与合并

### 边界情况处理

| 情况 | 处理方式 |
|------|---------|
| 空白幻灯片（无标题） | 归为"未分类"组，追加到末尾 |
| 某 PPT 只有封面 | 跳过该文件的内容页部分 |
| 同一 PPT 内重复标题 | 按原始顺序保留在同一组中 |
| 文件数量 < 2 | 报错提示至少需要 2 个文件 |
| 文件不存在或格式无效 | 报错提示具体哪个文件有问题 |
| 输出目录不存在 | 自动创建 |

## Skill 工作流

1. 用户提供多个 PPT 文件路径
2. Skill 验证所有文件存在且为有效 .pptx
3. 通过 Bash 执行 `python /Users/qqx/my_code_cursor/lq/scripts/ppt_merge.py <file1> <file2> ...`
4. 脚本执行合并，输出结果文件路径
5. 询问用户是否需要调用 ppt-restyle skill 统一样式

## 与 PPT 样式重塑 Skill 的关系

- 合并 skill 和 restyle skill 是独立的两个 skill
- 合并完成后，用户可选择是否调用 restyle skill 对合并结果统一应用模板样式
- 两个 skill 不存在硬依赖，可以各自独立使用

## 依赖

- Python 3.8+
- python-pptx（`pip install python-pptx`）
- lxml（python-pptx 自带）

## 已知限制

- 所有输入 PPT 必须使用相同的模板，否则版式引用可能不一致
- 不支持嵌入视频的幻灯片（将被复制但视频可能无法播放，发出警告）
- 幻灯片动画和切换效果按原样保留，不做特殊处理
- 标题归一化可能在极端情况下（如标题全为序号）产生误匹配
