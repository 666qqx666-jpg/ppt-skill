# PPT 样式重塑 Skill 设计文档

## 概述

一个 Claude Code skill，接收用户提供的 PPT 文件，将其所有内容保持不变，但将视觉样式替换为固定模板 PPT 的样式。

**技术栈：** python-pptx
**实现方案：** 模板克隆 + 占位符替换

## 模板结构

模板 PPT（`templates/template.pptx`）固定包含 2 页幻灯片：

1. **封面页（第1页）：** 包含背景 + 设计元素 + 一个标题占位符。转换时仅替换标题文本为源 PPT 封面的标题。
2. **内容页（第2页）：** 包含背景 + 设计元素 + 两个占位区域：
   - "页面标题" — 填入源幻灯片的标题
   - "页面内容" — 填入源幻灯片的所有内容元素

## 核心逻辑

### 输入/输出

- **输入：** 源 `.pptx` 文件路径（用户提供）
- **模板：** 从 `/Users/qqx/my_code_cursor/lq/templates/template.pptx` 读取
- **输出：** 保存至 `/Users/qqx/my_code_cursor/lq/output/<源文件名>_styled.pptx`

### 处理流程

1. **验证输入** — 检查源文件是否存在、模板是否存在、输出目录是否存在（不存在则创建）
2. **打开文件** — 用 python-pptx 加载源 PPT 和模板 PPT
3. **处理封面页：**
   - 克隆模板第1页
   - 提取源 PPT 第1页的标题文本
   - 将标题填入克隆页的标题占位符
4. **处理内容页（源 PPT 第2页至第N页）：**
   - 对每一页源内容页：
     a. 克隆模板第2页（内容布局）
     b. 提取源页标题 → 填入模板的"页面标题"占位符
     c. 识别源页上所有非标题元素（即"内容"部分）
     d. 计算源内容区域与模板内容区域的坐标映射
     e. 将每个元素复制到模板页的内容区域，调整位置
5. **保存** 新演示文稿到输出目录

### 按类型迁移元素

| 元素类型 | 迁移策略 |
|---------|---------|
| 文本框 / 带文本的自选图形 | 复制文本内容及段落级格式，重新映射位置坐标 |
| 图片 | 提取图片二进制数据，用 add_picture() 在映射坐标处添加，保持宽高比缩放至内容区域 |
| 表格 | 创建相同行列数的新表格，复制单元格文本、合并信息和基本格式，重新映射位置 |
| 图表 | 从源文件复制 chart part XML 到新幻灯片。python-pptx 对图表克隆支持有限，需要底层 lxml 操作 |
| 组合图形 | 递归复制子元素，重新映射组合的边界框，按比例调整子元素坐标 |
| SmartArt | 尝试 XML 级别复制。若失败，降级为图片并警告用户 |
| 其他图形（连接符、自由曲线等） | 深拷贝元素 XML，调整位置/尺寸属性，追加到新幻灯片的 spTree |

### 坐标映射

将源内容适配到模板内容区域的方法：

1. **模板内容边界：** 从内容占位符的位置属性获取（`left`、`top`、`width`、`height`）
2. **源内容边界：** 计算源幻灯片上所有非标题元素的外接矩形
3. **缩放系数：**
   - `scale_x = 模板宽度 / 源宽度`
   - `scale_y = 模板高度 / 源高度`
   - 取 `min(scale_x, scale_y)` 以保持宽高比
4. **逐元素映射：**
   ```
   new_left   = (shape.left - 源偏移x) × scale + 模板偏移x
   new_top    = (shape.top  - 源偏移y) × scale + 模板偏移y
   new_width  = shape.width  × scale
   new_height = shape.height × scale
   ```

### 边界情况处理

- **空白源页：** 跳过，不生成空白模板页
- **源 PPT 仅1页：** 输出仅包含样式化后的封面
- **内容超出模板区域：** 等比缩小以适配，不裁剪
- **模板占位符识别：** 优先使用占位符 idx 或名称匹配。若无法匹配，回退为基于位置检测（最大元素 = 内容区域）
- **字体处理：** 标题占位符使用模板字体；迁移的内容元素保留源字体

## Skill 定义（SKILL.md）

Skill 工作流程：
1. 接收源 PPT 文件路径作为参数
2. 通过 Bash 执行 `ppt_restyle.py`
3. 成功时返回输出文件路径
4. 失败时返回具体错误信息（模板缺失、PPT 格式无效等）

## 实现备注

- **幻灯片克隆：** python-pptx 没有内置 `clone_slide()` 方法。实现时需要深拷贝幻灯片 XML 元素，并在新幻灯片的关系集中重新注册所有关联资源（图片、图表等）。这是实现中最复杂的部分。
- **模板校验：** 启动时验证模板恰好有 2 页幻灯片，且每页都有可识别的占位符元素。不满足条件时立即报错退出。

## 依赖

- Python 3.8+
- python-pptx（`pip install python-pptx`）
- lxml（python-pptx 自带）

## 已知限制

- SmartArt 在 XML 复制失败时会降级为图片
- 不支持嵌入视频（将被跳过并发出警告）
- 含外部数据链接的图表会被转为内嵌数据（不保留外部链接）
- 源 PPT 的动画和切换效果不保留（使用模板的切换效果）
